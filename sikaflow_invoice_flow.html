Receipt vs Invoice, what changes and why

The current receipt is a proof-of-payment document issued after money changes hands. An invoice is a payment request, it comes before payment, includes a due date, customer details, tax breakdown, and the trader's bank/MoMo details so the buyer can pay later.

Implementation notes for Vietnam team — Phase 2 specification

🗄️ Data model — invoices table

New invoices table: id, invoice_number (auto-generated INV-YYYY-NNNN per MSME), msme_id, customer_name, customer_phone, issue_date, due_date, status (draft/sent/paid/cancelled/overdue), subtotal, tax_mts, tax_nhil, tax_getfund, total_due, paid_at, paid_via, notes. Plus a invoice_items child table with invoice_id, product_id, name, qty, unit_price, line_total.

🔢 Invoice numbering

Format: INV-{YYYY}-{NNNN} where NNNN is a zero-padded sequence per MSME per year (not global). Store invoice_sequences table: msme_id, year, last_seq. Use atomic UPDATE … RETURNING (same pattern as the programme reference number). Reset to 0001 on 1 Jan each year. Example: INV-2026-0047.

📤 Sharing — WhatsApp + link

Generate a public-facing invoice URL: sikaflow.app/inv/{invoice_id}. The page renders the invoice as HTML/CSS — no PDF library needed on the app. Sharing via WhatsApp opens the system share sheet with the URL pre-populated. The recipient sees a clean invoice in their browser without needing the app. Mark status as sent the first time the share button is tapped.

⏰ Overdue detection

Run a daily Celery beat task at 6 AM: query all invoices where status = 'sent' and due_date < today, update status to overdue. Send an SMS to the trader for each newly overdue invoice. The invoice list shows overdue invoices at the top with a red left border. Do not auto-send reminders to customers — trader controls this via the "Send Reminder" button.

💳 Payment details per MSME

Add a payment_details JSON field to the MSME profile — stores MoMo number, bank name, account number. Pre-populate from the registration data where available. These are shown on every invoice. The trader can edit them from Settings → Payment Details. Never expose these to the Vietnam team's codebase — they come from the MSME profile API endpoint.

🔗 Entry points

Two places to create an invoice: (1) Receipt screen → "Generate Invoice" button — pre-fills items from the just-completed sale. (2) Menu → Invoices → + button — blank form. The invoice form is a 3-step Flutter flow with a shared InvoiceProvider for state. Step 1: customer + dates. Step 2: items (re-uses existing product search widget). Step 3: payment details. Draft auto-saved on each step forward.
