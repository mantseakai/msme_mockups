Implementation notes for Vietnam team
📷 Photo capture
Use Flutter's image_picker package. Offer both camera capture and gallery pick. Compress to max 800×800 before upload. Store locally on device first; sync to backend when online. Show "Retake" button overlay on captured photo.
🔢 Quantity stepper
Replace the plain text input with a − / + stepper for quantity. Traders understand this instantly. Allow direct tap on the number to open a numeric keyboard for large quantities. Min value = 0, no max. Low-stock alert threshold defaults to 5, shown inline below the stepper.
📦 Advanced collapsible
Use Flutter ExpansionTile widget. Default state: collapsed. Tax type defaults to Taxable (write this server-side as the default, not from the UI). SKU and barcode are optional, do not mark with red asterisk. Barcode field has a "Scan" button that opens the camera scanner.
💰 Cost price & margin
Cost price is a new field, add cost_price DECIMAL to the products table. It is optional at this stage but required later for the Reports gross margin calculation. Label it "What did you pay for this?" not "Cost price" in the UI. Store it, do not display the margin calculation to the trader on this screen (Phase 3).
✅ Required fields
Only 3 required fields: Product name, Selling price, Quantity. Cost price and photo are strongly encouraged but not blocked. Validate on tapping Save, highlight the specific field in red, not a generic toast. The Save button stays active but shows errors inline on submit attempt.
🏷️ Label changes
Current Stock→How many do you have?
Low-stock Threshold→Alert me when below [N]
Selling Price→Selling price (What you charge)
[new field]→What did you pay for this?
