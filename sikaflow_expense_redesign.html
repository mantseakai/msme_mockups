Implementation notes for Vietnam team

🗄️ Data model — custom categories

Add a expense_categories table: id, msme_id (null = system default), name, icon_emoji, is_custom bool, created_at. System defaults have msme_id = null. Trader's custom categories have their msme_id. When loading the category grid, fetch WHERE msme_id IS NULL OR msme_id = :current_msme, ordered defaults first, then custom.

⚡ Bottom sheet behaviour

Use Flutter's showModalBottomSheet with isScrollControlled: true so the sheet can expand when the keyboard opens. Numpad is custom (not the system keyboard) — keeps layout predictable on all Sunmi screen sizes. Sheet dismisses on save or cancel. Back button on the phone closes the sheet, not the screen. Date defaults to today; no date picker needed in the sheet (keep it fast).

🔒 Category management rules

System default categories: read-only, cannot be edited or deleted by any trader. Custom categories: trader can rename or delete from a long-press menu on the tile. Deleting a category does not delete historical expense records that used it — they retain the category name as a string. Programme Admin portal: add a GET /admin/expense-categories/custom endpoint returning all custom categories grouped by MSME for reporting.

📋 Updated default category seed data

Seed the expense_categories table with these 11 defaults on deployment: Cost of Goods 🛍️, Market Levy / Tolls 🏛️, Rent / Stall Fees 🏪, Transport / Delivery 🚚, MoMo / Bank Charges 📱, Utilities 💡, Labour / Porter 👷, Packaging 📦, Phone / Airtime 📞, Equipment / Tools 🔧, Other ➕. The "Create new" tile is not a category row — it's a hardcoded UI action.

💡 Amount entry

Amount is required. Save button is disabled until amount > 0. Supplier name and note are optional — shown in the sheet but not blocking. Date is always today (no date picker in the quick sheet — trader can edit from the expense list if needed). Payment method defaults to Cash for now; can be added as a field in Phase 2. Write created_by_user_id from JWT on every expense record.

📊 Portal — custom category visibility

Portal must expose custom categories in reporting. Add a "Custom Categories" section to the Expenses report: total spend per custom category across all MSMEs, list of unique category names with count of MSMEs using each. This lets the programme team see what categories traders are creating (e.g. "Generator Fuel" appearing 40 times = add it to the default list next sprint).
