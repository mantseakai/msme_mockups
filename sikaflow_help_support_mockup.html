Implementation notes for Vietnam team

📱 WhatsApp link

Use the WhatsApp deep link format: https://wa.me/233XXXXXXXXX?text=Hi%20Sikaflow%20Support%2C%20I%20need%20help.%20My%20Business%20ID%20is%20SKF-ACC-1144. Pre-fill the message with the MSME's Business ID so support knows who is contacting them immediately. Trident Aliska to confirm the support WhatsApp number before go-live. This is a standard url_launcher call in Flutter no API needed.

❓ FAQ - local content

FAQ content is hardcoded in the app (not fetched from a server). Store as a JSON asset file: assets/faq.json with fields: category, question, answer. Use an ExpansionTile widget for accordion behaviour. Include a basic text search that filters questions client-side. Trident Aliska to supply the final question list before the sprint. The initial set from this mockup covers the most common support contacts from Field Agent feedback.

🐛 Report a problem

Collect: problem type (dropdown — 8 options), free text description (optional), device info (auto-collected from device). Use the device_info_plus package for model/OS. Submit to POST /api/v1/support/reports with all fields as JSON. Auto-attach last 500 lines of app log using a rotating log file (use logger package). Return a reference number (RPT-YYYYMMDD-NNNN) shown on the confirmation screen. Store reports in the backend support_reports table, admin portal shows them in a list.

🏷️ Menu tile change

Remove the disabled state from the Support tile. Replace the generic headset icon with the WhatsApp icon (ti-brand-whatsapp). Apply the green colour treatment: border: 2px solid #25D366, background: #f0fdf4, icon and label colour #166534. The label changes from "Support" to "Help & Support". Route the tap to HelpSupportScreen.

📋 Device info auto-fill

The device info strip on both the Help home and the Report form is populated at screen load — no user input needed. Fields: msme.programme_ref from auth state, app version from package_info_plus, device model + Android version from device_info_plus, network status from connectivity check. This information is also shown on the Help home screen so traders can read it out to support agents over the phone.

🌐 Language support

The WhatsApp pre-filled message, FAQ content, and problem type dropdown labels should all be translated to Twi and Ewe — the same as the rest of the app. The WhatsApp message template should be in the trader's selected language: e.g. Twi: "Mede wo ho brɛ Sikaflow support. Me Business ID ne SKF-ACC-1144." Trident Aliska to provide translations with the final FAQ content. Add a note to the WhatsApp message indicating the trader's language preference.
